h3. Hadoop 部署在 Linux 下，在 Windows 下用 Eclipse 远程连接 Hadoop 进行开发

h2. 平台环境

* Win7
* Eclipse Helios
** <a href="http://chxt6896.github.io/hadoop/2013/05/23/hadoop-eclipse-plugin.html" target="">Hadoop-eclipse-plgin</a>
* Hadoop-1.0.4 

h2. 目录结构

* eclipse目录 F:\eclipse
* eclipse工程目录 F:\workspace
* hadoop安装目录 F:\workspace\hadoop-1.0.4

h2. Step1 配置 Map/Reduce Locations

# 设置 Host 为要连接服务器的 IP 地址
# 设置 Advanced parameters 
** 修改 hadoop.tmp.dir 为连接 hadoop 服务器中设置的目录 
** 修改 dfs.replication 为连接服务器中 hdfs-site.xml 配置的值 

切换到 Map/Reduce 视图，查看是否已成功连接上服务器^^

h3. 当运行示例 WordCount 时会遇到两个问题

h2. Problem1

{% highlight text %}
13/05/26 20:18:05 WARN util.NativeCodeLoader: Unable to load native-hadoop library for your platform... using builtin-java classes where applicable
13/05/26 20:18:05 ERROR security.UserGroupInformation: PriviledgedActionException as:Administrator cause:java.io.IOException: Failed to set permissions of path: \tmp\hadoop-Administrator\mapred\staging\Administrator-519341271\.staging to 0700
Exception in thread "main" java.io.IOException: Failed to set permissions of path: \tmp\hadoop-Administrator\mapred\staging\Administrator-519341271\.staging to 0700
{% endhighlight %}

这个是 Windows 下文件权限问题，在 Linux 下可以正常运行，不存在这样的问题。

<b>解决方案</b>: 注释掉 hadoop-1.0.4/src/core/org/apache/hadoop/fs/FileUtil.java 里面 checkReturnValue 函数的内容（有些粗暴，在 Window 下可以不用检查）。重新<a href="http://chxt6896.github.io/hadoop/2013/05/26/hadoop-source-build.html" target="_blank">编译 Hadoop 源码</a>，重命名 hadoop-core-1.0.4-SNAPSHOT.jar 替换掉 hadoop-1.0.4 根目录下的 hadoop-core-1.0.4.jar 即可。
